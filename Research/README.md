# Research

Output folder for the Research agent. Every readout lives here as a dated markdown file.

## Naming
`YYYY-MM-DD-topic-slug.md`

Example: `2026-06-02-aesthetic-philosophies.md`

## What lives here
- Competitor teardowns
- Design reference scans
- AI / agentic pattern syntheses
- Article + transcript readouts
- Any external material that informs project decisions

## What does NOT live here
- Internal project notes — those live at root (`strategy.md`, `decision-log.md`, etc.)
- Daily working notes — `daily-log.md`
- Agent verdicts — `agents-log.md`

## Format
Every readout follows the format in `Agents/research.md` (TL;DR · Sources · Insights · Implications per agent · Recommended adaptations · Open questions).

## How readouts get acted on
1. Research saves the readout here.
2. Viet reads it.
3. Viet routes specific implications to the relevant agent ("Strategist, challenge the implication on anti-goals from `2026-06-02-aesthetic-philosophies.md`").
4. Relevant agent reviews through its critical path.
5. Viet decides whether to edit agent files / strategy / PRD.

Research never edits. Research informs.
